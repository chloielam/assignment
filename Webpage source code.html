Abstract:

Nowadays, “Open Source" and “Open Source License" are two words that are new to most beginner programmers. In this report, we discuss 3 main points: the definition of open source and open source license, the reason why an open source software still needs a license, and the information of 3 popular open source licenses which are GNU GPLv3, MIT License, Apache License 2.0.

In conclusion, an open source software is a software of which source code is publicly modified by everyone following a policy contract between the authors and the users, called an open source license. Each open source license’s preamble regulates different permissions, conditions, and limitations to please different organizations' purposes or needs and to maintain the order of an open source environment. GNU GPLv3 is considered as a strict license. Otherwise, MIT License is the most simple and most used one. Apache License 2.0 is less strict than GNU GPLv3, and it has a grant of patent rights compared to MIT License.

What is open source?

This term was introduced by Open Source Initiative in the late 1990s in the context of software development to designate a specific approach to creating computer programs. Open source commonly refers to a software people can together modify and share because it is designed to access publicly easily.

Nowadays, not just open source software, “open source” designates a broader set of values or also called “the open source way” like open source projects, or open source products which embrace and celebrate principles of open exchange, collaborative participation, rapid prototyping, transparency, meritocracy, and community-oriented development. But in this paper we will concentrate on open source in the context of computer programs: open source software.

Open source software is a software where its source code can be edited by everyone to enhance or enrich that software. We can understand that “source code” is a collection of coding lines written in a human-readable programming language that can manipulate how the software works. Therefore, if a person can access the source code, he/she can improve that program by adding features to it or fixing parts that don't always work correctly.

Examples:

Many current popular programs, platforms, or languages are considered as an open source: Linux operating systems, Android by Google, Firefox browser, Moodle, WordPress. Linux, for example, is the example par excellence of open source, and new programs can be created from its nucleus or kernel. Or the opening of Android permits any developer to create compatible applications. Consequently, thousands and thousands of applications exist to meet almost any need. Similarly, WordPress and Firefox are open source implies that there exists an enormous variety of compatible plug-ins that were independently developed.

What is an open source license?

Open source license can be simply understood as a legal contract between the software’s authors and the users which declares how that software is used in commercial applications under specified conditions.

Each open source license has different terms determining what the users can or cannot do with the software components and also their obligations. There are more than 200 different open source licenses with various complexity and requirements, so you can choose which one is compatible with your organization’s policies to maintain the compliance.

Open source license is mainly categorized into 2 kinds: copyleft and permissive. Copyleft licenses only allow the users to use, modify, and share the work as long as they still maintain the same kind of the opponent's license to the other users as well. Otherwise, with permissive licenses, the users can freely modify, and redistribute open source code, permitting its use in proprietary derivative works without requirement to use the same licences.

How does an open source license work?

Most of popular open source licenses are authorized by Open Source Initiative (OSI) which is a non-profit corporation with global scope formed to educate about and advocate for the benefits of open source, and to build bridges among different constituencies in the open source community.

An open source license would include 4 following statements:

  • Software can be modified, used commercially, and distributed.
  • Software can be modified, and used in private.
  • A license and copyright notice must be included in the software.
  • Software authors provide no warranty with the software and are not liable for anything.

For further details of each license description, visit this link: https://choosealicense.com/licenses/

Why does an open source software still need a license?

For those mentioned characteristics of open source software, open source license plays an important role in keeping an open source environment in order, friendly and legal. Using licenses can help the software owner to prevent some users’ illegal actions and ensure an organized environment for everyone to contribute to make the project enhanced. Also, it let the owners and the users define how the source code is used to support their further plans in the future.

GNU General Public License version 3 (GNU GPLv3)

GNU GPLv3 is a strong copyleft license of which detail terms shown in the Table above. For full preamble: https://choosealicense.com/licenses/gpl-3.0/

Usage:

As with the other license forms, at a minimum, a text file must be included within the root of the source code that contains the license document, and Free Software Foundation also recommends a boilerplate notification placed at the top or header section of each file within the software.

Acceptance:

Typically permitted for commercial use, modification, distribution, and patent grants. However, users/licensee/collaborators are not permitted to hold anyone liable, nor are they permitted to sublicense the work. Everyone is permitted to copy and distribute verbatim copies of the GPL license documents, but changing them is not allowed.

Examples:

Linux, Drupal, WordPress,...

MIT License

MIT License is a simple and short permissive license of which detail terms shown in the Table above. For full preamble: https://choosealicense.com/licenses/mit/

Usage:

Include a copy of it within the root of your source code or as a text file and include the current year and full name or names of the creators/authors.

Acceptance:

Typically the MIT License is suitable for commercial use, modification, distribution, and sublicensing. However, it is forbidden for licensees to hold the licensor/author liable for any issues or outcomes.

Examples:

JQuery, Ruby on Rails, Fedora,...

Apache License 2.0

Apache License 2.0 is also a permissive license of which detail terms shown in the Table above. Using Apache License 2.0, contributors can provide an express grant of patent rights compared to MIT License. For full preamble: https://choosealicense.com/licenses/apache-2.0/

Usage:

Must create a text file which is typically named “License” or “license.txt” for example, and it must be included in the root of the source code, and you must state any changes or version control as part of the code or documentation of the code. There is also a recommendation from the Apache Foundation that a boilerplate statement be placed within the header of each source file stating the nature of the Apache License.

Acceptance:

Most use cases of the Apache License are suitable for commercial use, modification, distribution, sublicensing, and patent grants. And like the previous license, users are forbidden from holding anyone liable; however, in addition, users are prohibited from attaching a trademark for any use of the project or software code.

Examples:

Android, The Apache Software Foundation, Subversion (SVN), NuGet,...

Evaluation:

Based on different conditions across various open source licenses, developers have a lot of options to choose the most compatible license to their project. Each licence will have its advantages and disadvantages. Therefore, both the developers and the users must consider the license’s preamble carefully before doing anything in order to avoid violating any illegal action.
